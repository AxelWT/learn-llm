## **从头开始训练因果语言模型**

以训练一个代码生成模型（如 CodeParrot）为例。

与之前的微调任务不同，这是从“随机初始化”或针对特定语料的大规模训练。

### 1. 总结

本节介绍了如何构建一个 **Decoder-only（仅解码器）**架构的模型。
其目标是让模型学会“接龙”，即根据给定的上文预测下一个词。
由于是从头训练，重点在于如何高效地准备大规模文本流，以及如何配置模型架构（如 GPT-2 或类似的 Transformer 结构）。

---

### 2. 处理数据集的流程

因果语言模型的数据处理核心是**将离散的文本变成连续的、固定长度的训练块**。

1. **数据流化（Streaming）：** 由于数据集（如代码库）通常极大，无法全部载入内存，因此使用 `iterable=True` 的方式流式读取数据。
2. **分词（Tokenization）：** 对原始代码或文本进行分词。
3. **串联（Concatenation）：** 将所有样本的 Token 连成一个极其漫长的序列，中间用 `EOS`（结束符）隔开。
4. **分块（Chunking/Grouping）：** * 按照模型支持的最大上下文长度（如 128、512 或 1024）将长序列切分成等长的“块”。
    * *优势：* 这种方法确保了每一个训练批次（Batch）都没有浪费，每个 Token 都在参与训练，没有多余的 Padding。
5. **生成标签：** 直接将 `input_ids` 复制一份作为 `labels`。

---

### 3. 如何计算损失值 (Loss)

CLM 的 Loss 计算基于**交叉熵损失（Cross-Entropy Loss）**，其本质是预测概率分布的准确性：

* **全序列计算：** 每一个位置产生的 Logits 都会参与计算。
* **计算目标：** 模型在位置 $t$ 的输出 Logits，要与位置 $t+1$ 的真实 Token 进行对比。
* **内部偏移：** 在 Hugging Face 的模型（如 `GPT2LMHeadModel`）内部，计算 Loss 时会自动将 `labels` 向左移动一位，将
  `input_ids` 向右移动一位，从而实现“用当前词预测下一个词”。

---

### 4. 选取什么数据作为训练时的参考 Labels

在因果语言模型中，**Labels（标签）就是输入数据本身。**

* **选取规则：** `labels = input_ids.copy()`。
* **物理意义：** 因为是自监督学习，文本中的每一个词既是当前的输入，也是前一个词的“正确答案”。

---

### 5. 训练过程中 “参考答案” 是什么

在训练的每一个瞬间，“参考答案”就是 **“序列中的下一个 Token”**：

1. **形态：** 与输入序列 `input_ids` 完全一致的整数向量。
2. **对应关系：**
    * 输入：`[Token 1, Token 2, Token 3]`
    * 模型预测：
        * 用 `Token 1` 预测 `Token 2`
        * 用 `Token 1 + Token 2` 预测 `Token 3`
    * **参考答案：** 对应位置上的 `Token 2` 和 `Token 3`。
3. **特殊处理：** * 与 MLM 或翻译不同，CLM 通常不需要在 Labels 中填入大量的 `-100`，因为序列中的每个词都是有效的学习目标。
    * 只有在极少数需要填充（Padding）的情况下，填充位才会被设为 `-100` 以被 Loss 函数忽略。

### 核心差异对比表

| 任务类型               | 核心模型架构           | Labels 来源     | “参考答案”的本质         |
|:-------------------|:-----------------|:--------------|:------------------|
| **标记分类 (7.2)**     | Encoder          | 外部人工标注        | 类别索引 (如 NER 标签)   |
| **掩码建模 (7.3)**     | Encoder          | 原始输入          | 被遮盖位置的“原词”        |
| **翻译/摘要 (7.4-5)**  | Encoder-Decoder  | 目标文本 (Target) | 目标语言/摘要的 Token 序列 |
| **从头训练 CLM (7.6)** | **Decoder-only** | **原始输入**      | **序列中的下一个词**      |

CLM 正是目前大语言模型（如 GPT-4, Llama 3）最基础的训练逻辑。
