# 自回归模型详解

## 定义

**自回归（Autoregressive）** = 用前面的词预测后面的词，一个接一个，像多米诺骨牌一样逐个生成。

## 核心思想

```
预测当前词 → 只看它前面的词（已经生成的）
```

## 数学表达

$$
P(w_1, w_2, ..., w_n) = P(w_1) \times P(w_2|w_1) \times P(w_3|w_1,w_2) \times ... \times P(w_n|w_1,...,w_{n-1})
$$

每个词的概率只依赖前面已经出现的词。

## 自回归生成过程示例

```
步骤1: 输入 "今天"
       → 模型预测概率最高的词: "天气"

步骤2: 输入 "今天天气"
       → 模型预测: "很好"

步骤3: 输入 "今天天气很好"
       → 模型预测: "。"

最终生成: "今天天气很好。"
```

## 与掩码语言模型的对比

| 类型 | 代表模型 | 预测方式 | 适用场景 |
|------|----------|----------|----------|
| **自回归** | GPT 系列 | 从左到右，看前面预测后面 | 文本生成、对话 |
| **掩码模型** | BERT 系列 | 遮住中间词，看前后预测中间 | 文本理解、分类 |

## 可视化对比

### 自回归（GPT）

```
"The weather is ____"
   ✓      ✓     ✓      ?
  已知   已知   已知    预测

特点: 只看左边，单向
```

### 掩码模型（BERT）

```
"The ____ is nice"
   ✓      ?     ✓    ✓
  已知   遮住   已知  已知

特点: 看两边，双向
```

## 类比理解

| 模型类型 | 类比 | 特点 |
|----------|------|------|
| **自回归** | 写作 | 一个字接一个字写，只看前面已写的内容 |
| **掩码模型** | 填空题 | 同时看前后，猜中间缺失的内容 |

## 自回归的优缺点

### 优点

- 自然语言生成能力强（写文章、对话、故事）
- 训练目标简单清晰
- 生成过程自然流畅

### 缺点

- 只能看左边，不能看右边（单向注意力）
- 不适合理解整句语义（BERT 双向更适合）
- 生成速度较慢（必须逐词生成）

## 常见自回归模型

| 模型 | 参数规模 | 特点 |
|------|----------|------|
| GPT-2 | 1.5B | 早期经典自回归模型 |
| GPT-3 | 175B | 强大的文本生成能力 |
| GPT-4 | 多模态 | 支持图像理解 |
| Claude | 大规模 | 对话能力强 |

## 训练过程

```
原始文本: "今天天气很好，适合出门散步"

自回归训练:
- 输入: "今天"        → 预测: "天气"
- 输入: "今天天气"    → 预测: "很好"
- 输入: "今天天气很好" → 颐测: "，"
- ...

每一步: 最大化预测下一个词的概率
```

## 一句话总结

```
自回归 = 像写作一样，一个词接一个词往下写，只看前面，不看后面
```

# 语言模型分类详解

## 一、按预训练目标分类

这是与自回归、掩码模型同一维度的分类方式。

### 1. 自回归模型（Autoregressive, AR）

| 项目 | 内容 |
|------|------|
| **代表模型** | GPT 系列 |
| **核心思想** | 看前面，预测后面，单向生成 |
| **数学形式** | $P(w_1,...,w_n) = \prod P(w_i|w_1,...,w_{i-1})$ |
| **适用场景** | 文本生成、对话、故事创作 |

```
"The weather is ____"
   ✓      ✓     ✓      ?
  已知   已知   已知    预测

特点: 单向注意力，只能看左边
```

### 2. 自编码/掩码模型（Autoencoding, AE）

| 项目 | 内容 |
|------|------|
| **代表模型** | BERT、RoBERTa、DistilBERT |
| **核心思想** | 遮住部分词，看前后猜中间 |
| **数学形式** | $P(w_{mask}|w_{context})$，双向理解 |
| **适用场景** | 文本理解、分类、情感分析、问答 |

```
"The ____ is nice"
   ✓      ?     ✓    ✓
  已知   遮住   已知  已知

特点: 双向注意力，能看两边
```

### 3. 混合模型（AR + AE）

| 项目 | 内容 |
|------|------|
| **代表模型** | XLNet、ERNIE |
| **核心思想** | 结合自回归和掩码的优点 |
| **技术方法** | 排列语言模型（PLM） |
| **适用场景** | 通用 NLP 任务 |

**XLNet 的排列语言模型**：
```
原始词序: [w1, w2, w3, w4]

排列1: [w3, w1, w4, w2] → 按此顺序自回归预测
排列2: [w2, w4, w1, w3] → 不同排列顺序预测

效果: 通过不同排列，间接实现"看两边"
```

### 4. 前缀语言模型（Prefix LM）

| 项目 | 内容 |
|------|------|
| **代表模型** | UniLM |
| **核心思想** | 前面双向理解，后面单向生成 |
| **适用场景** | 翻译、摘要、问答生成 |

```
任务：翻译 "The cat is cute" → "这只猫很可爱"

输入部分（前缀）: [The cat is cute] ← 双向注意力，能看前后
输出部分: [这只猫很可爱] ← 单向注意力，只能看前面（自回归）

理解原文 → 生成译文
```

### 5. 去噪自编码器（Denoising AE）

| 项目 | 内容 |
|------|------|
| **代表模型** | BART、T5 |
| **核心思想** | 破坏文本，学会还原 |
| **破坏方式** | 词序打乱、删除、替换、遮盖 |
| **适用场景** | 翻译、摘要、文本修复 |

```
原始文本: "今天天气很好，适合出门散步"

破坏后: "今天天气很好散步出门适合"
        ↑ 词序被打乱

模型任务: 还原成原始文本
输出: "今天天气很好，适合出门散步"

架构: 编码器（双向理解） + 解码器（自回归生成）
```

### 预训练目标对比总结

| 模型类型 | 代表模型 | 注意力方向 | 擅长任务 |
|----------|----------|------------|----------|
| **自回归** | GPT | 单向（左→右） | 生成 |
| **掩码** | BERT | 双向 | 理解 |
| **混合** | XLNet | 双向（通过排列） | 通用 |
| **前缀** | UniLM | 前缀双向 + 后缀单向 | 理解+生成 |
| **去噪** | BART、T5 | 编码双向 + 解码单向 | 理解+生成 |

## 二、按架构分类

| 架构类型 | 代表模型 | 特点 | 擅长什么 |
|----------|----------|------|----------|
| **仅编码器** | BERT、RoBERTa | 双向理解，不能生成文本 | 分类、理解 |
| **仅解码器** | GPT 系列 | 单向生成，不做双向理解 | 写作、对话 |
| **编码器-解码器** | T5、BART | 先理解再生成 | 翻译、摘要 |

```
仅编码器（BERT）:
输入 → [编码器] → 理解表示
       双向↑
       输出是向量表示，不是文本

仅解码器（GPT）:
输入 → [解码器] → 生成文本
       单向↑
       每次只能看前面已生成的内容

编码器-解码器（T5）:
输入 → [编码器] → [解码器] → 输出文本
       双向↑        单向↑
       先理解输入    再生成输出
```

---

## 三、按应用场景分类

| 类型 | 代表模型 | 擅长什么 | 典型任务 |
|------|----------|----------|----------|
| **理解型** | BERT | 分析、判断 | 分类、情感分析、问答抽取 |
| **生成型** | GPT | 创作、对话 | 写文章、聊天、续写故事 |
| **理解+生成型** | T5、BART | 转换、加工 | 翻译、摘要、改写、纠错 |

```
理解型任务: "这句话是正面还是负面情感？"
生成型任务: "续写这篇文章..."
理解+生成: "把这篇长文章缩写成50字摘要"
           先理解原文 → 再生成摘要
```

## 四、按模型规模分类

| 级别 | 参数量 | 代表模型 | 特点 |
|------|--------|----------|------|
| **轻量级** | ~60M | DistilBERT、TinyBERT | 速度快，适合部署 |
| **中小型** | ~110M | BERT-base | 平衡性能与效率 |
| **中型** | ~340M | BERT-large | 性能更强 |
| **大型** | ~1.5B | GPT-2 | 生成能力强 |
| **超大型** | ~175B | GPT-3 | 接近人类水平 |
| **巨大型** | 更大 | GPT-4、Claude | 多模态、强推理 |

---

## 五、按训练数据分类

| 类型 | 代表模型 | 训练方式 | 特点 |
|------|----------|----------|------|
| **纯文本预训练** | GPT、BERT | 大量无标注文本 | 基础能力 |
| **多模态预训练** | GPT-4、CLIP | 文本+图像 | 理解图片 |
| **指令微调** | ChatGPT | 预训练+指令数据 | 听懂人类指令 |
| **强化学习微调** | GPT-4、Claude | 预训练+RLHF | 更安全、有用 |

```
训练流程演进:
纯预训练 → 指令微调 → RLHF
  ↓           ↓         ↓
懂语言    听懂指令   更安全有用
```

## 六、五大分类维度总结

| 分类维度 | 类别 |
|----------|------|
| **预训练目标** | 自回归 / 掩码 / 混合 / 前缀 / 去噪 |
| **架构** | 仅编码器 / 仅解码器 / 编码器-解码器 |
| **应用场景** | 理解型 / 生成型 / 理解+生成型 |
| **规模** | 轻量 / 中型 / 大型 / 超大型 |
| **训练数据** | 纯文本 / 多模态 / 指令微调 / RLHF |

## 七、一句话记住各种模型

```
自回归（GPT）     → 单向，写作生成
掩码模型（BERT）  → 双向，理解分析
混合模型（XLNet） → 两者优点结合
前缀模型（UniLM） → 前面理解，后面生成
去噪模型（BART）  → 破坏再还原，理解+生成

仅编码器（BERT）  → 只理解，不生成
仅解码器（GPT）   → 只生成，不做双向理解
编解码器（T5）    → 先理解再生成，翻译摘要
```
