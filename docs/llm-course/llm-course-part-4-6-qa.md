## **问答系统**

本章介绍了如何处理 SQuAD 这种**抽取式问答（Extractive QA）**任务，即模型需要从一段给定的上下文中精准地“掐头去尾”，找回答案的起始和结束位置。

---

### 1. 总结

本节重点解决了将原始文本标签（字符位置）转换为模型可学习标签（Token 位置）的工程难题。核心在于处理**超长上下文**：
当一段文字超过模型最大长度（如 512）时，使用“滑动窗口”将其切分成多个重叠的块，并确保模型在每个块中都能正确寻找答案，或者识别出答案不在该块中。

---

### 2. 处理数据集的流程

问答任务的预处理是所有 NLP 任务中最复杂的之一，流程如下：

1. **分词与截断（Tokenization with Sliding Window）：**
    * 使用 `AutoTokenizer` 对“问题”和“上下文”进行编码。
    * 设置 `truncation="only_second"`（只截断上下文）并启用 `return_overflowing_tokens=True`。
    * 利用 `stride`（步长）创建**滑动窗口**，使长文本被切分为多个有重叠的特征（Features）。

2. **获取偏移映射（Offset Mapping）：**
    * 启用 `return_offsets_mapping=True`。这会记录每个 Token 在原始字符串中的起始和结束字符索引。

3. **确定答案的 Token 边界：**
    * 利用 `sequence_ids` 区分哪些 Token 属于问题，哪些属于上下文（答案只会在上下文中）。
    * 对比原始数据中的答案字符位置与 `offset_mapping`，找到答案对应的第一个 Token 和最后一个 Token 的索引。

4. **处理“无答案”情况：**
    * 如果答案的边界超出了当前滑动窗口块的范围，则将该块的起始和结束位置都标记为 `0`（通常对应 `[CLS]` 标签）。

---

### 3. 如何计算损失值 (Loss)

在抽取式问答中，模型被设计为拥有两个“分类头”：一个预测起点，一个预测终点。

* **计算机制：** 采用 **交叉熵损失（Cross-Entropy Loss）**。
* **计算公式：**
  对于序列中的每一个位置，模型输出两个 Logits。损失值是“起点分类”和“终点分类”损失的算术平均值：
  $$Loss = \frac{CrossEntropy(start\_logits, start\_positions) + CrossEntropy(end\_logits, end\_positions)}{2}$$
* **训练逻辑：** 模型试图最大化正确起始 Token 和正确结束 Token 的概率，而不关心其他 Token。

---

### 4. 选取什么数据作为训练时的参考 Labels

在 QA 任务中，`labels` 并不是文本，而是**两个整数索引**：

* **`start_positions`**：答案在当前 Token 序列中开始位置的索引。
* **`end_positions`**：答案在当前 Token 序列中结束位置的索引。

---

### 5. 训练过程中 “参考答案” 是什么

“参考答案”在训练时是一组**指针（Pointers）**：

1. **具体形态：** 它告诉 Loss 函数，在 $N$ 个 Token 组成的序列里，哪一个是正确答案的“头”，哪一个是“尾”。
2. **特殊情况的处理：**
    * **答案完整在块内：** 参考答案是真实的 Token 索引（例如：Start=45, End=48）。
    * **答案跨块或不在块内：** 参考答案被设为 **`0`**。这意味着模型被教导：如果在这个窗口里找不到答案，就去指向 `[CLS]`。
3. **预测与反馈：**
   模型输出两个长度为 $L$（序列长度）的向量。训练时，它会将这两个向量与上述的“指针索引”进行对比。如果模型在非答案位置给出了高分（Logit），Loss
   就会升高。

---

### 💡 核心差异提醒

这一章的 QA 与上一章的翻译/摘要最大的不同在于：

* **翻译/摘要（Seq2Seq）：** 是“生成”新的文本，参考答案是目标语言的 Token 流。
* **问答系统（Extractive）：** 是“提取”已有文本，参考答案是**位置坐标**。这就是为什么 QA
  训练通常比生成任务快得多的原因——它只需要做两次多分类，而不是漫长的文本接龙。
