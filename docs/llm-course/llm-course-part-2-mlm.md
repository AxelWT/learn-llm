## **微调一个掩码（mask）语言模型**

这是自监督学习的核心，通常用于让模型适配特定领域的文本（如医学、法律或特定的影评语境）。

---

### 1. 总结

本节介绍了如何对预训练的掩码语言模型（以 `distilroberta-base` 为例）进行**领域适配（Domain Adaptation）**。
模型通过随机“遮盖”输入序列中的部分 Token，并尝试根据上下文预测这些被遮盖的内容，从而学习该特定领域的语言特征和表达方式。

---

### 2. 处理数据集的流程

MLM 的数据处理逻辑非常高效，因为它不需要人工标注，标签是“自动生成”的。

1. **分词（Tokenization）：** 对原始文本（如 IMDb 影评）进行分词，此时不涉及任何标签。
2. **分块（Grouping/Chunking）：** * 将所有样本拼接在一起，形成一个巨大的 Token 流。
    * 按照模型支持的最大长度（如 128 或 512）切分成固定长度的块。
    * *目的：* 确保训练时每个 Batch 都是满载的，提高计算效率。
3. **动态遮盖（Dynamic Masking）：**
    * 这一步通常在 **`DataCollatorForLanguageModeling`** 中完成，而不是在预处理脚本中固定死。
    * 每一轮训练（Epoch），模型会随机选择序列中 **15%** 的 Token 进行处理。

---

### 3. 如何计算损失值 (Loss)

MLM 使用的是典型的**交叉熵损失（Cross-Entropy Loss）**。

* **计算范围：** 损失**仅针对被选中的那 15% 的 Token** 进行计算。对于没有被选中的其余 85% 的 Token，模型不计算它们的
  Loss（即不产生梯度）。
* **计算目标：** 模型在被遮盖的位置会输出一个针对全词表的概率分布，Loss 的目标是让模型给“正确的原始 Token”分配最高的概率。

---

### 4. 选取什么数据作为训练时的参考 Labels

在代码层面，`labels` 最初是 `input_ids` 的完整副本。

* **选取规则：** 在数据处理阶段，我们直接执行 `labels = input_ids.clone()`。
* **修正逻辑：** 随后，我们会根据掩码策略，将那些**没有被选为预测目标**的 Token 在 `labels` 中修改为 **`-100`**。

---

### 5. 训练过程中 “参考答案” 是什么

“参考答案”（Reference Answer）在 MLM 训练中具有一种“镜像”特性：

1. **形态：** 它是与 `input_ids` 维度完全一致的整数张量。
2. **内容：** * **在被遮盖位置（被替换为 `[MASK]`、随机词或保持不变的位置）：** 参考答案就是该位置**原本的词 ID**。
    * **在非预测位置：** 参考答案是 **`-100`**。
3. **遮盖策略（80-10-10 规则）：**
   为了让模型学会不仅仅是依赖 `[MASK]` 标记，在选中的 15% 目标中：
    * 80% 真的被替换为 `[MASK]`。
    * 10% 被替换为一个随机的 Token。
    * 10% 保持原样不变。

   **关键点：** 无论这 15% 的 Token 变成了什么，它们的“参考答案”始终是它们**最初的样子**。

**一句话总结：** 参考答案就是被模型“遮蔽”或“篡改”之前的原始 Token ID。
